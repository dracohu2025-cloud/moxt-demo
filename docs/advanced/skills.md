# 把你的思维模型，变成 momo 的 Skill

每个人思考问题的方式都不一样。

有人习惯先拆解假设，有人习惯先问用户，有人遇到任何问题都会先画一张图。这些不是"方法论"，而是你在长期工作中磨出来的直觉——它们是你判断力的一部分。

momo 可以学会这些。当你把自己的思维模型告诉 momo，它会把它整理成一个 Skill，成为它工作方式的一部分。从那以后，它不只是在执行你的指令，而是在用你的方式思考。

---

## 几个例子

### 创始人：遇到分歧，先打碎假设

有位创始人特别警惕"大家都觉得应该这样"这种感觉。他的习惯是，每当讨论陷入分歧，先问：我们在哪个假设上不一致？他把这个思路告诉了 momo，变成了一个"假设拆解"Skill。现在每次他把会议纪要扔给 momo，momo 会自动标出讨论中隐含的、未说出口的假设——省掉了大量澄清工作。

### 律师：每个结论都要有出处

有位律师有一个原则：任何判断，必须能指向具体条款或先例。她把这个习惯告诉了 momo——每次 momo 帮她整理法律意见时，会自动标注每条结论的依据出处，没有来源的会提醒她补充或核实。

### 投资人：先问"最坏的情况是什么"

有位早期投资人每次看项目，第一个问题不是"它能长多大"，而是"它会怎么死"。他把这个习惯整理成了一个"死亡情景"Skill。现在 momo 帮他分析任何新项目时，都会先列出三个最可能导致失败的路径，再谈机会。

### 医生：永远先排除最危险的可能

有位临床医生有一个诊断习惯：不管症状多普通，先在脑子里快速过一遍"最坏的可能是什么"，排除掉才往下走。他让 momo 学会了这个顺序——每次帮他梳理病例信息时，momo 会优先列出需要排查的高风险情况，而不是直接给出最可能的诊断。

### 教师：先问学生"你觉得呢"

有位老师相信，真正的理解来自学生自己说出来，而不是听来的。她从不直接给答案，总是先反问一句"你觉得呢"。她把这个习惯告诉了 momo，整理成"苏格拉底式追问"Skill。现在她让 momo 帮学生批改作业时，momo 不会直接指出错误，而是先提问，引导学生自己发现问题。

### 设计师：凡事先画出来再说

有位设计师无论讨论什么——流程、系统、用户路径——都习惯先画一张示意图，哪怕很草。她觉得语言容易产生误解，视觉对齐最快。她让 momo 学会了这个习惯：每次讨论涉及流程或结构时，momo 会主动生成一份可视化草图，作为对话的起点。

### 记者：每个信源都要交叉验证

有位记者有一条铁律：一个信源说的事，不算事实。她把这个原则整理成了 momo 的 Skill。现在 momo 帮她整理调研资料时，会自动标出哪些信息只有单一来源，哪些已经有多方印证——她不需要每次手动过滤。

### 厨师：先想清楚"这道菜的灵魂是什么"

有位主厨在设计每一道新菜之前，都会先问自己：这道菜的核心味道是什么，其他所有元素都是为它服务的。他把这个框架告诉了 momo，整理成"核心定义优先"Skill。现在他让 momo 帮他整理菜单方案或食材搭配时，momo 会先追问核心风味定位，再展开其他建议。

---

## 怎么把你的思维模型变成 Skill

你不需要写得很系统。只需要告诉 momo：

> "我有一个习惯，每次做决策之前，我都会先列出三个最坏的可能结果。你可以把这个整理成一个 Skill，以后帮我做决策分析的时候自动用这个框架。"

momo 会帮你整理成一个结构化的 Skill，保存在你的 Workspace 里。下次它再帮你工作，就不只是按你说的做——而是按你的方式想。

**共事越久，它越接近你。**
