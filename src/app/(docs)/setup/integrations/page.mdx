# 接入外部工具

Moxt 的核心能力之一是让 momo 连接你们已有的工具。接入越多，momo 能做的事越多。

---

## Slack

接入之后，两件事变得可能：

**在 Slack 里直接给 momo 发指令**

> @momo 把这周的用户反馈汇总一下

不用打开 Moxt，在 Slack 里说一句话，结果直接出现在 Workspace 里，Slack 里同步推送摘要。

**让 momo 监测 Slack 里的信息**

你可以让 momo 定时扫描指定频道，把重要信息提炼出来——用户反馈频道、竞品讨论频道都适合。

**接入方式**：在 AI 同事或 momo 的对话框上方，点击 Slack 图标即可连接。也可以通过设置 → 集成 → Slack 进入。

---

## GitHub

接入 GitHub 之后，momo 就能同时读懂你们的产品文档和代码——它能做的事会发生质变。

运营可以让它从代码里读出功能细节写帮助文档。PM 可以让它对比代码和 PRD 找出实现偏差。新人入职不需要问人，直接问 momo 架构是怎么设计的。

### 接入步骤

**第一步：授权**

> **需要 GitHub 管理员权限**：接入组织仓库时，操作者必须是该 GitHub Organization 的 **Owner 或 Admin**。如果你没有管理员权限，请让团队里有权限的人来完成这一步。

1. 在 Moxt 左侧边栏，点击「设置」→「集成」→「GitHub」
2. 点击「连接 GitHub」
3. 选择授权方式：
   - **个人账号授权**：接入你个人账号下的仓库
   - **组织授权**：接入团队组织下的仓库（推荐，需要组织管理员权限）
4. 授权完成后回到 Moxt

**第二步：选择仓库**

授权完成后，选择你想要接入的仓库。建议先接入最核心的一到两个，不需要一次全部接入。

**第三步：验证**

接入完成后，去问 momo 一个关于代码的问题，比如：

> "你能看到我们的 GitHub 仓库吗？简单描述一下这个项目的技术架构。"

### 接入之后你能做什么

**让运营自己读懂代码**

> "从我们的 GitHub 代码里，找出注册流程的限制条件（比如用户名格式、密码要求），整理成一段用户看得懂的说明。"

**新人入职速通**

> "我刚加入这个团队，帮我读一下 GitHub 仓库，用非技术语言解释一下这个产品的核心模块是什么，各自负责什么。"

**发现 PRD 和代码的偏差**

> "对比一下 Workspace 里的注册流程 PRD，和 GitHub 里实际的代码实现，有没有明显的不一致？"

**让 momo 直接改代码、提 PR**

> "帮我把登录页的报错信息从英文改成中文，改完直接提一个 PR。"

momo 会 clone 仓库、创建分支、修改代码、提交 PR，你只需要 review 和合并。

## 通用集成（API Key + HTTP）

这是最灵活的接入方式。

**思路**：存一个 API Key，写一段说明告诉 momo 怎么调。momo 执行时自动取密钥、发请求。Mixpanel、BigQuery、Linear、飞书……任何有 HTTP API 的服务都适用。

**典型场景**：

- 让 momo 从 Mixpanel 拉数据生成周报
- 用飞书 API 自动发通知
- 从 BigQuery 查询用户行为数据回答运营问题

**接入方式**：设置 → 集成 → 密钥管理，添加 API Key。然后创建一个 Skill，用自然语言描述 API 的调用方式（endpoint、认证方式、常用操作），momo 就能自动发现并使用。

## MCP（外部工具协议）

MCP 让 momo 像装插件一样获得工具能力——不是调 API 拼请求，而是工具的能力直接变成 momo 的技能。

**典型场景**：

- 接入 Sentry MCP → 直接问「最近有什么线上报错？」，momo 像你自己打开 Sentry 一样查询
- 接入 Linear MCP → 「帮我创建一个 bug ticket，优先级 high」
- 接入 Figma MCP → 「这个设计稿里的按钮用了什么颜色？」

**接入方式**：直接告诉 momo「帮我连接 Sentry」，它会引导你提供 Token，自动完成配置。也可以在设置 → 集成 → MCP 中手动管理。

---

## Webhook

让外部系统主动通知 Moxt——Sentry 报了错、CI 部署完了、表单收到新提交——都可以通过 Webhook 推过来，momo 自动处理。

**典型场景**：

- Sentry 报错 → momo 自动分析堆栈，如果影响用户超过 100 人，通知 #engineering
- CI 部署完成 → momo 自动跑一轮冒烟测试检查
- Typeform 收到新提交 → momo 自动归类并更新用户反馈汇总

**接入方式**：设置 → 集成 → Webhook → 创建新 Webhook。用自然语言写一段「处理指令」告诉 momo 收到事件后做什么，Moxt 会生成 URL 和 Token，填到外部系统里即可。

---

## 小结

| 接入方式 | 适合场景 | 上手成本 |
| -------- | -------- | -------- |
| Slack / GitHub | 高频工具，开箱即用 | 低 |
| 通用集成 | 任何有 API 的服务，最灵活 | 中 |
| MCP | 让 momo 像装插件一样获得工具能力 | 中 |
| Webhook | 外部系统主动推事件，momo 自动处理 | 低 |

不确定该选哪种？直接问 momo，它会推荐最适合的方式。
