# 文件格式与导入说明

Moxt 使用三种核心文件格式，这不是偏好，是架构决策——AI 读写这些格式的效率最高，token 消耗低，生成准确率高。

---

## 三种核心格式

**Markdown（.md）— 文档的格式**

适合：文章、报告、会议纪要、产品文档、策略备忘录等所有文字内容。

Markdown 是纯文本，AI 可以直接读写，不需要解析富文本格式。你在 Moxt 里创建的所有文档，默认都是 Markdown。

**CSV（.csv）— 结构化数据的格式**

适合：用户数据、业务指标、表格信息等所有结构化数据。

AI 可以直接对 CSV 执行统计分析，写代码处理它，生成可视化报告——比 Excel 更直接。

**HTML（.html）— 富展示的格式**

适合：交互式看板、可视化报告、演示文稿等需要视觉呈现的内容。

Moxt 里的 HTML 文件是真实可交互的，可以包含图表、筛选器、动态数据。momo 可以生成这类文件，你直接在 Workspace 里打开就能看。

---

## 导入时的格式转换

你从其他地方带来的文件，Moxt 会自动转换：

| 来源格式 | 转换为 | 说明 |
| -------- | ------ | ---- |
| Word (.docx) | Markdown | 格式基本保留，复杂表格可能需要手动调整 |
| PDF | Markdown | 文字内容提取，图片单独保存 |
| Google Docs | Markdown | 通过 Google Drive 集成导入 |
| Notion 页面 | Markdown | 通过 Notion 导出 zip 包导入 |
| Notion Database | CSV | 字段和数据保留，视图不保留 |
| Excel (.xlsx) | CSV | 数据保留，格式和公式不保留 |

---

## 哪些格式不支持

- 图片（.png、.jpg 等）：不支持单独的文件，只能存在文档中
- 音视频：只能作为附件保存在文档中
- .exe、.dll 等二进制文件：不支持上传

---

## 文件大小限制

| 项目 | 限制 |
| ---- | ---- |
| 单个文件 | 最大 50MB |
| 批量上传 | 单次最多 100 个文件 |
| 图片 | 建议压缩至 5MB 以下以保证加载速度 |

---

## 文件命名建议

Moxt 的 AI 会读文件名来理解文件内容。建议：

- 用清晰的中文或英文描述文件内容，不要用「文档 1」「副本」这类无意义的名字
- 用状态前缀区分文档阶段：`[WIP]`（进行中）、`[归档]`（已过时归档）
- 文件名中有日期时，用 `YYYY-MM-DD` 格式，便于排序
